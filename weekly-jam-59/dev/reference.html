How to Use

Variable Tweak File

The variable tweak file contains a number of engine variables that can be modified at run-time for easy tweaking, without recompiling.
The variables present in the file and what they modify are as follows:

refresh_rate -- The rate at which the game updates and renders.
window_title -- The title of the window.
window_start_width -- The starting width of the window.
window_start_height -- The starting height of the window.
viewport_width -- The width of the actual screen render contents (also the minimum width of the window).
viewport_height -- The height of the actual screen render contents (also the minimum height of the window).
border_clear_color -- The color to fill the area outside the viewport with at the start of each update.
viewport_clear_color -- The color to fill the viewport area with at the start of each update.
mixer_frequency -- The frequency to play-back sounds and music (requires restart to take effect).
mixer_channels -- The number of audio channels, mono, stereo, etc.(requires restart to take effect).
mixer_chunksize -- The quality and latency of sound play-back (requires restart to take effect).
mixer_tracks -- The number of unique tracks that sounds can play on.
mixer_volume -- The current volume to play-back auido.
console_mini_height -- The height of the console when in its small state.
console_large_height -- The height of the console when in its large state.
console_left_padding -- How far from the left to begin drawing text.
console_bottom_padding -- How far from the bottom to begin drawing text.
console_input_color -- The color to draw the current input text line in.
console_history_color -- The color to draw the history text lines in.
console_back_color -- The color to draw the console background in.
console_scroll_speed -- The speed at which the console moves between states (set to 0 for instant snapping).
console_cursor_blink_interval -- The interval at which the console cursor blinks at.
enable_hotloading -- Whether to use the asset hotloading system or not.
enable_debug_console -- Whether to use the debug console system or not.

Font File Format

Font files can be found in the "data/fonts/" directory.
The custom file format for loading in monospace bitmap fonts for the game to use:

glyph_size -- The size of the glyphs that make up the font.
glyph_cache -- The texture used to represent the various glyphs (loaded from the directory "data/textures/".

An example font file can be seen below:

# This is a comment!
glyph_size 8 16
glyph_cache "console.png"

This creates a font using the "console.png" glyph-set with each glyph being 8x16 pixels in size.

Animation File Format

Animation files can be found in the "data/animations/" directory.
The custom file format for loading in frame-by-frame animations for the game to use:

texture_offset -- The offset in the texture file where the animation will begin clipping frames.
frame_size -- The size of each of the frames that make up the animation.
looped -- A flag stating whether the animation should loop or not.
time -- How long a key frame should play for, in seconds. Each instance of the time keyword in the file is a new key frame for the animation.

An example of an animation file can be seen below:

# This is a comment!
texture_offset 0 0
frame_size 0 0

looped true

time 0.3
time 0.3
time 0.3
time 0.3
time 0.3
time 0.3
time 0.3

This creates a 7-frame looped animation with frames starting at position (0,0) of size (100,100) where each frame is 0.3 seconds long.
Frames are clipped from left-to-right in a straight line from the texture_offset position.

If you are wondering how to associate an animation with a texture, that is handled in code. See the API function: TCE.RenderAnimatedTexture()

Main Lua Scrupt

The "data/scripts/main.lua" file is the core of any game written in the engine. This is the file that the engine looks for as a start-point and to execute.
The basic template for the main.lua file is as follows:

local TCE = require("data.scripts.tce.tce")

function Initialize ()
 -- ...
end

function Update (_dt)
 -- ...
end

function Render ()
 -- ...
end

After loading the main script the game will execute the Initialize() function to setup the game. Then the engine will then proceed to call Update() and Render() for evey cycle of the game loop.
The line local TCE = require("data.scripts.tce.tce") at the top of the file loads the engine's API into local scope for faster access of all the functions, constants, etc. The API will always be accessed through this local table.
The _dt variable passed to the Update() function is the frame's delta time, passed from the engine. The game can use this to ensure proper frame-independent timing and logic.

Some important things to take note of when programming in the main Lua script are as follows:
 1.) The main Lua script is hotloaded so saving any changes made whilst running the game will update in real-time.
 2.) The Initialize() function is only called once at start-up and, subsequent hotloads will not call it again.
 3.) No global/persistent state should be stored within the main Lua script itself!
The reasoning for points 2.) and 3.) are because of the fact that the file is hotloaded.
If Initialize() were to be called upon saving changes then the game would reset each time, making issues difficult to debug.
And if persistent state were to be stored in the main Lua script then it would be overwritten when the file is reloaded.

The solutions for issues 2.) and 3.) are as follows:
 2.) [NOTE: Currently not implemented! Soon there will be a feature to do a reload of the cache when the developer wants, without needing to close and reopen the game!]
 3.) Store the game state globally in other script file(s) that are then required by the main Lua script. This would look like so:

[[ data.scripts.state ]]

my_entity = { x=0, y=0 }

[[ data.scripts.main]]

local TCE = require("data.scripts.tce.tce")
require(data.scripts.game_state)
function Initialize ()
 -- ...
end

function Update (_dt)
 -- my_entity.x = (do some calculation)
 -- my_entity.y = (do some calculation)
end

function Render ()
 -- ...
end

This way the global game state remains persistent but the update and render logic can be dynamically reloaded.

Debug Console

Basic Usage

The debug console can be used to run various debug commands to make testing the game easier.
The console can be turned on using ` this will open it in a small state. Pressing it again will make the console large, and once more will close it.

The console can be controlled using the following inputs:

Left -- Move the cursor to the left.
Right -- Move the cursor to thr right.
Up -- Cycle up through the history.
Down -- Cycle down through the history.
Escape -- Clear the current input.
Enter -- Enter the current input.
Backspace -- Delete character before the cursor.
Delete -- Delete character after the cursor.

Adding Console Commands

Console commands can be added by modifying the Console module and adding new function, new name of the function is the name of the command.
An example of adding a console command is as follows:

function Console.add(a, b)
 print(a + b)
end

This console command could then be executed by writing add 10 20 in the console.

Detailed API Reference

Texture

texture_handle = TCE.LoadTextureFromFile(filename)

Loads a texture with the name filename from the "data/textures/" directory and returns a texture handle that can be used to reference the texture in other functions.
This function can load the following image formats: JPEG, PNG, BMP, PSD, TGA, GIF, HDR, PIC, PPM, and PGM.

TCE.RenderTexture(texture_handle, { x, y, [scale_x], [scale_y], [angle], [anchor_x], [anchor_y], [flip] })

Renders the texture referenced by texture_handle at position x, y; with the optional transformation values applied.
scale_x and scale_y define how much to scale the texture by.
angle specifes how much to rotate the texture by (in radians).
anchor_x and anchor_y specify the point on the texture to rotate around (these values are automatically scaled if scale_x or scale_y are set).
flip specifies how to flip the texture, it can be one of: TCE.RenderFlips.NONE, TCE.RenderFlips.HORZ, TCE.RenderFlips.VERT, TCE.RenderFlips.BOTH.

TCE.RenderAnimatedTexture(texture_handle, animation_handle, { x, y, [scale_x], [scale_y], [angle], [anchor_x], [anchor_y], [flip] })

Works the same as TCE.RenderTexture(), however it takes an animation referenced by animation_handle and uses it to animate the texture.

TCE.SetTextureColor(texture_handle, color_string)

Sets the color of the texture referenced by texture_handle to the color specified in color_string. This color will remain until subsequent calls.
A color string is simply a string formatted into a 32-bit number represeting an RGBA color, e.g: "0xFF7FAAFF".
Note that if a color string is formatted in hexadecimal then it must be prefixed with "0x".
A color string can also be made by combining separate RGBA integer values using TCE.MakeColor().

texture_width = TCE.GetTextureWidth(texture_handle)

Gets the width of the texture referenced by texture_handle.

texture_height = TCE.GetTextureHeight(texture_handle)

Gets the height of the texture referenced by texture_handle.

Animation

animation_handle = TCE.LoadAnimationFromFile(filename)

Loads an animation with the name filename from the "data/animations/" directory and returns an animation handle that can be used to reference the animation in other functions.

TCE.PauseAnimation(animation_handle, pause)

Sets whether the animation referenced by animation_handle should be paused or not.
If pause is true then the animation will be paused, otherwise it will be unpaused.

paused = TCE.AnimationIsPaused(animation_handle)

Returns true if the animation referenced by animation_handle is paused and false if not.

TCE.ResetAnimation(animation_handle)

Resets an animation to go back to its first frame. Allowing for non-looped animations to be played multiple times when needed.

frame_width = TCE.GetAnimationFrameWidth(animation_handle)

Gets the frame width of the animation referenced by animation_handle.

frame_height = TCE.GetAnimationFrameHeight(animation_handle)

Gets the frame height of the animation referenced by animation_handle.

Font

font_handle = TCE.LoadFontFromFile(filename)

Loads a font with the name filename from the "data/fonts/" directory and returns a font handle that can be used to reference the font in other functions.

TCE.RenderText(font_handle, text, { x, y, [scale_x], [scale_y], [angle], [anchor_x], [anchor_y], [flip] })

Renders the string text using the font referenced by font_handle starting at position x, y; with the optional transformation values applied.
scale_x and scale_y define how much to scale the font glyphs.
angle specifes how much to rotate the each glyph by (in radians).
anchor_x and anchor_y specify the point on the glyphs to rotate around (these values are automatically scaled if scale_x or scale_y are set).
flip specifies how to flip the glyphs, it can be one of: TCE.RenderFlips.NONE, TCE.RenderFlips.HORZ, TCE.RenderFlips.VERT, TCE.RenderFlips.BOTH.

TCE.SetFontColor(font_handle, color_string)

Sets the color of the font referenced by font_handle to the color specified in color_string. This color will remain until subsequent calls.
A color string is simply a string formatted into a 32-bit number represeting an RGBA color, e.g: "0xFF7FAAFF".
Note that if a color string is formatted in hexadecimal then it must be prefixed with "0x".
A color string can also be made by combining separate RGBA integer values using TCE.MakeColor().

glyph_width = TCE.GetFontGlyphWidth(font_handle)

Gets the glyph width of the font referenced by font_handle.

glyph_height = TCE.GetFontGlyphHeight(font_handle)

Gets the glyph height of the font referenced by font_handle.

text_width = TCE.GetTextWidth(font_handle, text)

Gets the resulting width that the string text will take up when rendered using the font referenced by font_handle.

text_height = TCE.GetTextHeight(font_handle, text)

Gets the resulting height that the string text will take up when rendered using the font referenced by font_handle.

Sound

sound_handle = TCE.LoadSoundFromFile(filename)

Loads a sound with the name filename from the "data/sounds/" directory and returns a sound handle that can be used to reference the sound in other functions.
This function can load the following sound formats: WAV.

TCE.PlaySound(sound_handle, [loops])

Plays the sound referenced by sound_handle.
If loops is specifed, or greater than zero, then the sound will loop that many times before stopping.
If loops is set to -1 then the sound will continuously play.
[WARNING: Currently there is no way to stop infinitely looping sounds!]

Event

x, y = TCE.GetMousePosition()

Gets the current mouse position within the window and stores it in x, y.
The x value will be between 0 and TCE.GetViewportWidth().
The y value will be between 0 and TCE.GetViewportHeight().

x, y = TCE.GetRelativeMousePosition()

Gets how much the mouse has moved since mouse motion was last polled and stores it in x, y.
The x value will store how much the mouse moved horizontally.
The y value will store how much the mouse moved vertically.

x, y = TCE.GetGlobalMousePosition()

Gets the current mouse position on the desktop and stores it in x, y.
The x value will be between 0 and the current display width.
The y value will be between 0 and the current display height.

key_down = TCE.IsKeyDown(keycode)

Returns true if keycode is down and false if it is up.
keycode can be any of the values in TCE.Keycodes.
If keycode is not a valid key then the function will always return false.

key_up = TCE.IsKeyUp(keycode)

Returns true if keycode is up and false if it is down.
keycode can be any of the values in TCE.Keycodes.
If keycode is not a valid key then the function will always return false.

key_pressed = TCE.KeyPressed(keycode)

Returns true if keycode has been pressed and false if not.
A pressed event is fired once as soon as the key is pressed and will not fire again until the key has been released and pressed again.
This is different from TCE.IsKeyDown() which fires every frame whilst the key is down.
keycode can be any of the values in TCE.Keycodes.
If keycode is not a valid key then the function will always return false.

key_released = TCE.KeyReleased(keycode)

Returns true if keycode has been released and false if not.
A released event is fired once as soon as the key is released and will not fire again until the key has been pressed and released again.
This is different from TCE.IsKeyUp() which fires every frame whilst the key is up.
keycode can be any of the values in TCE.Keycodes.
If keycode is not a valid key then the function will always return false.

button_down = TCE.IsButtonDown(button)

Returns true if button is down and false if it is up.
button can be any of the values in TCE.Buttons.
If button is not a valid button then the return value is undefined.

button_up = TCE.IsButtonUp(button)

Returns true if button is up and false if it is down.
button can be any of the values in TCE.Buttons.
If button is not a valid button then the return value is undefined.

button_pressed = TCE.ButtonPressed(button)

Returns true if button has been pressed and false if not.
A pressed event is fired once as soon as the button is pressed and will not fire again until the button has been released and pressed again.
This is different from TCE.IsButtonDown() which fires every frame whilst the button is down.
button can be any of the values in TCE.Buttons.
If button is not a valid buttion then the return value is undefined.

button_released = TCE.ButtonReleased(button)

Returns true if button has been released and false if not.
A released event is fired once as soon as the button is released and will not fire again until the button has been pressed and released again.
This is different from TCE.IsButtonUp() which fires every frame whilst the button is up.
button can be any of the values in TCE.Buttons.
If button is not a valid buttion then the return value is undefined.

Primitive

TCE.RenderPoint(x, y, color_string)

Renders a point at position x, y in the color specified by color_string.
A color string is simply a string formatted into a 32-bit number represeting an RGBA color, e.g: "0xFF7FAAFF".
Note that if a color string is formatted in hexadecimal then it must be prefixed with "0x".
A color string can also be made by combining separate RGBA integer values using TCE.MakeColor().

TCE.RenderLine(x1, y1, x2, y2, color_string)

Renders a line from position x1, y1 to x2, y2 in the color specified by color_string.
A color string is simply a string formatted into a 32-bit number represeting an RGBA color, e.g: "0xFF7FAAFF".
Note that if a color string is formatted in hexadecimal then it must be prefixed with "0x".
A color string can also be made by combining separate RGBA integer values using TCE.MakeColor().

TCE.RenderRectOutline(x, y, w, h, color_string)

Renders a rectangle outline at position x, y of size w, h in the color specified by color_string.
A color string is simply a string formatted into a 32-bit number represeting an RGBA color, e.g: "0xFF7FAAFF".
Note that if a color string is formatted in hexadecimal then it must be prefixed with "0x".
A color string can also be made by combining separate RGBA integer values using TCE.MakeColor().

TCE.RenderRectFilled(x, y, w, h, color_string)

Renders a filled rectangle at position x, y of size w, h in the color specified by color_string.
A color string is simply a string formatted into a 32-bit number represeting an RGBA color, e.g: "0xFF7FAAFF".

Window

viewport_width = TCE.GetViewportWidth()

Gets the width of the viewport (ignoring upscaling based on window size).
The viewport is the visible area of the window where the game is rendered.

viewport_height = TCE.GetViewportHeight()

Gets the height of the viewport (ignoring upscaling based on window size).
The viewport is the visible area of the window where the game is rendered.

TCE.ShowCursor(show)

TCE.Exit()

Gracefully exits the game.

Collision

collided = TCE.PointAndBoxCollision(x, y, { x, y, w, h })

Returns true if the point x, y is within the bounds of box { x, y, w, h } and false if not.

collided = TCE.BoxAndBoxCollision({ x, y, w, h }, { x, y, w, h })

Returns true if the either of the { x, y, w, h } boxes overlap/collide and false if not.

Utility

color_string = TCE.MakeColor(r, g, b, a)

Takes the values provided by r, g, b, a and converts them into a color string for use with various engine functions.
A color string is simply a string formatted into a 32-bit number represeting an RGBA color, e.g: "0xFF7FAAFF".
The color string can be converted back into separate RGBA components using TCE.MakeRGBA().
Note that if the r, g, b, a values are not between 0-255 they will under/overflow.

r, g, b, a = TCE.MakeRGBA(color_string)

Takes the color string color_string values provided and converts them into a the separate components r, g, b, a.
A color string is simply a string formatted into a 32-bit number represeting an RGBA color, e.g: "0xFF7FAAFF".
The resulting r, g, b, a values will be within 0-255.
The separate RGBA components can be converted back into a color string using TCE.MakeColor().
